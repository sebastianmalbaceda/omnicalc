# Arquitectura Full-Stack: Tablas de Combos Maestros (Edición Profesional 2026)

> **Cómo leer estas tablas:**
>
> - Todas las opciones de cada capa están en una sola columna. No hay una única opción correcta — depende del proyecto, el equipo y el contexto.
> - El símbolo ⭐ marca la opción que encaja mejor en la **mayoría de proyectos estándar, profesionales y estables** de 2026.
> - Las combinaciones que se usan juntas como una sola unidad aparecen como `Tecnología A + Tecnología B`.

**Indicadores de coste y licencia:**

- 🆓 Gratuito / Open Source (autoalojable sin coste de licencia)
- 🔀 Freemium (tier gratuito funcional + planes de pago para escalar)
- 💰 De pago / Comercial (requiere suscripción o pago por uso en producción real)

Stripe Elements para formularios embebidos, Stripe SDK para integración server-side. Control total, cumplimiento normativo estricto (SOC 2, ISO 27001), escala prácticamente ilimitada. Requiere equipo DevOps dedicado. La elección cuando el proyecto o la empresa lo exige. **Vercel + Railway ⭐:** La combinación más práctica para proyectos profesionales nuevos en 2026. Vercel para frontend (Next.js o estático); Railway para backend y base de datos. **Hetzner VPS + Coolify:** La opción más económica y sin vendor lock-in. Ideal para proyectos con tráfico estable que quieren evitar costes variables y dependencia de plataformas. **Render:** Alternativa directa a Railway, similar en filosofía, precios y facilidad de uso.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              | Infraestructura donde viven el frontend, el servidor y la base de datos.             |

Playwright (Microsoft, open source) cubre Chromium, Firefox y WebKit — más rápido y estable que Cypress. Vitest es compatible con la API de Jest pero mucho más rápido al integrarse con Vite. **Cypress + Jest:** Muy extendido en proyectos existentes. Cypress tiene tier gratuito pero el dashboard CI es de pago. Válido para mantener, no la primera elección en proyectos nuevos. **Playwright + Jest:** Si ya se usa Jest como runner unitario pero se quiere modernizar los tests E2E con Playwright.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      | Pruebas automáticas que verifican que el código no se rompe.                         |

Ecosistema de plugins maduro (type-aware rules, react-hooks, accesibilidad). **Biome + Husky:** Binario único en Rust, entre 10 y 100× más rápido que ESLint + Prettier. Excelente para proyectos nuevos sin necesidad de plugins ESLint específicos. Caveat: el ecosistema de plugins de Biome v2 aún está madurando. **ESLint + Biome (híbrido):** Si se necesita la velocidad de Biome para formateo pero se mantienen plugins específicos de ESLint que Biome aún no cubre. Husky bloquea commits en todos los casos. Todas las opciones son gratuitas y open source.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           | Formateo automático de código y detección de errores de estilo antes de cada commit. |

Tier gratuito funcional para proyectos pequeños; self-hosted open source sin coste. Indica la línea exacta de código que falló con contexto completo del usuario y entorno. **Datadog:** Monitorización de infraestructura completa (APM, logs, métricas). De pago, orientado a enterprise con presupuesto. **PostHog:** Si se necesitan analíticas de producto (eventos, funnels, feature flags) además del rastreo de errores. Open source y self-hostable. **GlitchTip:** Alternativa 100% gratuita y open source a Sentry, compatible con su SDK. Para equipos que quieren monitorización sin ningún coste de servicio.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | Alertas y trazabilidad de errores en producción.                                     |

Mayor complejidad a cambio de máximo control visual. **React Native Paper:** Si se prefiere seguir las guías de Material Design de Google.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          | Sistemas de diseño que funcionan igual en móvil y web.                    |
